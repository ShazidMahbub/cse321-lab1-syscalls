iAbout this course lab

Over the semester this lab covers the core building blocks of an operating system through practical exercises: the Linux command line and C programming, process and thread management, thread synchronization, inter-process communication, and the internals of xv6, a small instructional kernel in which you implement system calls and process scheduling. The emphasis throughout is on understanding each concept by implementing it.

iThis week: System Calls & Process Creation

A system call is the interface a program uses to request a service from the operating system. This session covers the file and file-descriptor calls (open, read, write, lseek, close), the process calls (fork, exec, wait), and how to trace the calls a program makes. The sections that follow are interactive: you compile and run C in a terminal, operate file descriptors directly, and build process trees step by step rather than only reading about them.

03 · System calls

What Is a System Call?

A system call is a request for the operating system to perform an action on a program's behalf, such as accessing a file or creating a process: operations a program cannot carry out on its own.

Why they existA program runs in user space with limited privileges. Accessing hardware, files, or the process table requires the kernel, and the system call is the controlled entry point to it.

The ruleEvery fork() doubles the process count, so after n forks there are 2ⁿ processes, and a printf() after k forks runs in all 2ᵏ of them. For example, printf("A"); fork(); printf("B"); prints one “A” and two “B”. This is the basis for Practice Task 3.

⌗After fork(), memory is not shared

Try it Separate address spaces

⟂Order is not guaranteed — race it

Run it Parent & child output interleaving

Run it several times — without wait(), parent and child output can interleave differently each run. Turn wait() on to force the child to finish first.

●C
#include <sys/types.h>
#include <unistd.h>
#include <stdio.h>

int main() {
    pid_t pid = fork();

    if (pid == 0)
        printf("\nI'm the child process\n");
    else if (pid > 0)
        printf("\nI'm the parent process. My child pid is %d\n", pid);
    else
        perror("error in fork");

    return 0;
}
06 · wait, exec & process states

wait(), exec() & Zombies

wait() synchronizes a parent with its children, and exec() replaces a process's program entirely. The state machine below shows how a process moves from running to terminated to reaped.

⏳wait() and zombies

Zombie processA process that has terminated but whose exit status hasn't been collected by its parent. It lingers in the process table (its PID can't be reused) until the parent calls wait(). Reproduce it in the Process lab (Section 5): fork → child exit() → parent hasn't waited.
Step it Orphan vs zombie — when a child outlives its parent

Press Next to watch a child get orphaned and reparented to init.

●C
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    int status;
    pid_t pid = fork();
    if (pid == -1) { printf("fork failed\n"); exit(1); }

    if (pid == 0) {            /* Child */
        printf("Child here!\n");
    } else {                   /* Parent */
        wait(&status);
        printf("Well done kid!\n");
    }
    return 0;
}

⟳The exec family — click to compare

Explore execl / execv / execlp / execvp

Common mistakeThe path must be the absolute path to the executable, for example /bin/ls, not a relative path or a directory with a trailing slash. Arguments are double-quoted strings ("a"), not single-quoted characters ('a'). Note also that a successful exec does not return.
●C
int main() {
    printf("1\n");
    pid_t pid = fork();
    if (pid == 0)
        execl("/bin/ls", "ls", NULL);     /* child runs ls */
    else if (pid > 0)
        execl("/bin/pwd", "pwd", NULL);    /* parent runs pwd */
    return 0;
}
07 · Worked examples

Run the Examples

Step through each program to see the process tree, file, and output develop. The explanation can be revealed afterwards.

1Nested fork loop

●C
#define FORK_DEPTH 3

int main() {
    int i, r;
    pid_t my_pid = getpid();

    for (i = 1; i <= FORK_DEPTH; i++) {
        r = fork();
        if (r > 0) {
            printf("Parent process %d forked child process %d\n", my_pid, r);
        } else if (r == 0) {
            my_pid = getpid();
            if (i == FORK_DEPTH) {
                r = execl("/bin/echo", "/bin/echo", "Hello World", NULL);
                exit(1);   /* never expected to reach here */
            }
        } else { exit(1); }   /* r < 0: fork failed */
    }
    return 0;
}
Step it Watch the tree grow & "Hello World" print
1processes
0loop i
0“Hello World”
Program output

Start: just P0. Each iteration, every process forks once — so the count doubles.

What happenedNeither the parent nor a still-looping child breaks out, so every process forks once per iteration: 1 → 2 → 4 → 8. The 4 children born at i = 3 reach FORK_DEPTH and execl("/bin/echo", …), so "Hello World" prints 4 times. After a fork, i is the same in parent and child; a child updating my_pid does not change the parent's copy. /bin/echo has the same PID as the child that exec'd; code after a successful execl never runs.

2open / write / close — run it in the File lab

●C
char teststr[] = "The quick brown fox jumps over the lazy dog.\n";   /* 45 bytes */

int main() {
    int fd = open("testfile", O_WRONLY | O_CREAT, 0600);
    if (fd < 0) { perror("File open failed"); exit(1); }
    printf("Attempting to write %d bytes\n", strlen(teststr));   /* 45 */
    ssize_t r = write(fd, teststr, strlen(teststr));            /* writes 45 */
    printf("Wrote %d bytes\n", (int) r);
    close(fd);
    return 0;
}
What happenedIt creates (or opens) testfile with permission 0600, writes the 45-byte string, and prints Attempting to write 45 bytes then Wrote 45 bytes. Besides O_WRONLY you can open with O_RDONLY or O_RDWR. open returns a non-negative file descriptor on success, or -1 on failure.

3Two writes with lseek — the “50, not 90” one

●C · snippet
r   = write(fd, teststr, len);   /* 1st write: bytes 0..44  -> file is 45 bytes */
off = lseek(fd, 5, SEEK_SET);    /* move the offset back to byte 5 */
r   = write(fd, teststr, len);   /* 2nd write: bytes 5..49  -> file is 50 bytes */
close(fd);
What happenedThe first write fills bytes 0..44 (file = 45). lseek(fd, 5, SEEK_SET) moves the offset back to byte 5, so the second write overwrites bytes 5–44 and extends the file only to byte 49 — final size 50 bytes, not 90. Besides SEEK_SET there are SEEK_CUR and SEEK_END.

4chdir then execl — step it

●C
int main() {
    int r = chdir("..");                       /* go up one directory */
    if (r < 0) { perror("Eek!"); exit(1); }
    r = execl("/bin/ls", "/bin/ls", NULL);     /* becomes ls, lists the parent dir */
    perror("Double eek!");   /* only runs if execl fails */
    return 0;
}

08 · Practice problems

Practice — Yours to Write

Each task includes progressive hints. Attempt the problem first, then reveal the hints one at a time. The solution itself is left for you to write.

1Task 1 — append strings to a file

Statement. Open a file given as a command-line argument, then repeatedly ask the user for strings to write to it. Keep going until the user enters -1. Create the file if it doesn't exist.
Hint 1The filename is argv[1] (check argc >= 2). Open with O_WRONLY | O_CREAT | O_APPEND, mode 0644.
Hint 2Loop: read a line with fgets, compare with strcmp(buffer, "-1") (mind the trailing newline), break when it matches.
Hint 3Write each accepted string with write(fd, buffer, strlen(buffer)); close(fd) after the loop.

2Task 2 — ordered grandchild / child / parent

Statement. Create a child and a grandchild. Each prints a line, but force the output order: I am grandchild, I am child, I am parent.
Build it Force the order with wait()

Step through: the deepest process prints first because each ancestor wait()s for its descendant before printing.

Hint 1Nest the forks: parent forks the child; inside the child's branch, fork again for the grandchild.
Hint 2Use wait() so each ancestor waits for its descendant before printing.
Hint 3Grandchild prints immediately; child wait()s then prints; parent wait()s then prints.

3Task 3 — odd-PID forks & counting

Statement. Given a=fork(); b=fork(); c=fork();, check each child's PID and if it is odd, that process forks again. Print how many processes were created.
Hint 1Three sequential forks alone make 2³ = 8 processes — use the fork program simulator in Section 5 to see it.
Hint 2Capture each fork's return: pid = fork(); if (pid > 0 && pid % 2 == 1) { fork(); }.
Hint 3Memory isn't shared after fork — a counter is copied, not shared. Have each process print one line (with getpid()) and count the lines.

4Task 4 — sort.c, oddeven.c, and a split

Statement. Write sort.c (prints args sorted descending) and oddeven.c (prints odd/even for each arg). Then a program where the child sorts the array and the parent prints odd/even.
Hint 1Read numbers with atoi(argv[i]) for i = 1 .. argc-1 into an int array.
Hint 2Declare the array in the parent, fork(), put sorting in the child branch and odd/even in the parent branch. wait() so output doesn't interleave.
Hint 3To exec the helpers instead, convert ints to strings and use execv with a char *argv[] array (easier than execl for variable counts).

5Task 5 — one child, three grandchildren

Statement. The parent creates one child and three grandchildren, and prints their IDs.
Build it Construct the required tree

Goal: parent → child → three grandchildren. Only the child forks the grandchildren — and they must not keep forking.

Hint 1The parent forks once for the child. The child forks three times. Make sure grandchildren do not keep forking.
Hint 2In the child: for (i=0;i<3;i++) { pid=fork(); if (pid==0) { printf(...); break; } } — the break stops each grandchild spawning more.
Hint 3Use getpid() / getppid() to print IDs, and wait() for clean ordering.

The concepts
  • System call — asking the OS to do something privileged.
  • open / read / write / lseek / close — the file I/O workflow.
  • fork() — creates a child; returns twice (0 / child PID).
